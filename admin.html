Maintainer Data Editor

Edit source data in the browser, create a Git branch through the GitHub API, and open a pull request without leaving the static site.

Maintainer-only page

This page is intended for repository maintainers. Your GitHub fine-grained personal access token is used only in the browser. Use the minimum repository permissions, prefer session-only storage, and clear the token when you finish.

Authentication

The editor reads local site data anonymously, then uses the GitHub REST API only when you test the token or create a pull request.

Warzone schedule editor

Edits data/manual-schedules.json. Source and notes fields help document the PR but are not written into the JSON contract.

Valid orders: "", 1-2-3, 1-3-2, 2-1-3.

Tracked market item editor

Reads data/market/items/items.csv and updates data/market/items/tracked_items.json. Category and notes are PR metadata only. Enabled controls whether the item remains in the tracked list.

Open house editor

Directly edits data/open-houses.json. Existing source fields stay intact, and manual entries can use source type manual.

Pull request workflow

The editor creates a branch named maintainer-update-YYYYMMDD-HHMMSS, commits only changed source files, and opens a PR against main.
