Maintainer Data Editor

Edit source data in the browser and commit durable source files straight to main without leaving the static site.

Maintainer-only page

This page is intended for repository maintainers. Your GitHub fine-grained personal access token is used only in the browser. Use the minimum repository permissions, prefer session-only storage, and clear the token when you finish.

Authentication

The editor reads local site data anonymously, then uses the GitHub REST API only when you test the token or commit source changes to main.

Warzone schedule editor

Edits data/manual-schedules.json. Source and notes fields help document the commit but are not written into the JSON contract.

Valid orders: "", 1-2-3, 1-3-2, 2-1-3.

Tracked market item editor

Reads data/market/items/items.csv and updates data/market/items/tracked_items.json. Category and notes are commit metadata only. Enabled controls whether the item remains in the tracked list.

Open house updates

data/open-houses.json is regenerated from GitHub issues by scripts/update_open_houses.py. Browser edits are disabled here so maintainers do not create changes that the rebuild workflow will overwrite.

Submit new listings with the repository open-house issue template. Use the maintenance issue template for edits or removals so the generated registry stays durable.

Direct Commit Workflow

The editor validates source changes, writes only changed durable source files straight to main, and lets the existing GitHub Actions workflows regenerate public outputs afterward.
